The AI/ML side:
This code scrapes the rss news of the listed financial institutions (banks) and financial news sites.
Then, sorts the text according to categories and sub-categories, summarises the news article into a shorter form (by BART).
Then, all the news articles within the sub-category are combined into one text chunk and summarised into a sub-category-wide summary (by Gemma-3-27b).
Then, the shorter form (by BART) is classified under 'title' (because in a prior version the variable was used for something else and changing it to Summary would be time consuming), the Gemma-3 summary, BART summary, categories, sub-categories, feed links, original articles are packaged into a dictionary.
Then, an overall summary for all of the news articles are created (executive summary).
Then, the executive summary, a dataframe of all articles (df_all) and the aforementioned dictionary are exported into 3 .pkl files for the flask website to import.

->The index/homepage displays the results of processing using AI/ML. NOTE: the article's "title" (which is actually the BART summary as mentioned) is displayed under Summary in the HTML and the "summary" (which is actually the Original article) is displayed under Original Article.

->The /gpt_qna and /predict pages allow access to Gemma 3-27b and the previously scraped articles inside df_all. First, the keyword inserted inside "Ingest" textbox is used to find all articles (within df_all) with the keyword, the articles and the question inserted in prompted question are both fed into Gemma 3-27b model. The creativity is adjustable. All of this is done with prompt engineering and occurs within the prompt. The model will be prompted with the articles and question, once with a prompt suggesting the model should give a concise summary and once with a prompt suggesting the model should reply with a longer executive summary. The responses are displayed.

->The /pandas and /ask pages allow the user to insert a url and after using newspaper4k to convert the url into text, Gemma 3-27b shall read the text, reply with a string consisting of all numerical values detected and the label. The reply would be cleaned and displayed in a plotly bar chart. Optional choices: If the World bank option is selected, the Gemma 3-27b model will be given 3 separate decisions, once to choose relevant world bank indicator sources, once to choose a list of countries and once to choose relevant indicators (after the choice of sources is used to create a list of available indicators for each source). The indicators would be checked for the possibility of being archived or deleted and must have at least one row of data before being visualised in a plotly line graph. The slider allows the user to limit the number of data sources chosen. If the Description is selected, an interpretation of the line chart will be generated by Gemma 3-27b.

Usually the process of converting a news website (if newspaper4k could infact download it) to bar chart without the additional options is: 10-20 seconds. With world bank charts, it is usually 19 - 24 seconds. With chart description, it is usually 40 - 60 seconds (as the Gemma 3 model would need to read an entire dataframe and the article)

->The /find and /search pages allow the user to use DuckDuckGo to scrape news websites for a keyword, aggregate the descriptions and summarise them into a concise abstract, a detailed explanation and strategies (using Gemma 3-27b). There are options regarding the aggregation process like modifying the initial sample size used to determine the main categories/topic and number of articles scraped per category to investigate deeper. There are the same options with world bank charts.

Usually the process of scraping news articles' descriptions and summarising them takes: 48 - 66 seconds. With charts, it takes around 120 seconds. With charts and description: takes 240 seconds. Expectedly, the higher amount for sample size and scraped articles, the longer it takes.
